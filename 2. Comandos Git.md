
# 1. Navegación y Sistema

Estos comandos te permiten moverte por las carpetas de tu computadora y entender dónde estás ubicado.

| **Comando**   | **Qué hace**                                                                             |
| ------------- | ---------------------------------------------------------------------------------------- |
| `pwd`         | _(Print Working Directory)_ Te dice en qué carpeta estás ahora mismo.                    |
| `ls`          | _(List)_ Te muestra todos los archivos y carpetas de donde estás.                        |
| `ls -R`       | _(List Recursive)_ Muestra archivos y carpetas de forma recursiva (incluye subcarpetas). |
| `cd <nombre>` | Entra en una carpeta. Usa comillas si tiene espacios: `cd "mi carpeta"`.                 |
| `cd ..`       | Sale de la carpeta actual y te sube un nivel hacia atrás.                                |
| `cd ~`        | Te lleva a la carpeta raíz de tu usuario (el origen).                                    |
| `git version` | Te dice qué versión de Git tienes instalada.                                             |
| `Ctrl + C`    | Corta cualquier proceso que se esté ejecutando en la terminal.                           |

---
# 2. Gestión de Archivos y Carpetas

Comandos para crear, mover, borrar o editar contenido directamente desde la consola.

| **Comando**        | **Qué hace**                                                            |
| ------------------ | ----------------------------------------------------------------------- |
| `mkdir <nombre>`   | Crea una carpeta nueva.                                                 |
| `mkdir -p <a/b/c>` | Crea múltiples carpetas anidadas (una dentro de otra).                  |
| `touch <archivo>`  | Crea un archivo vacío (ej. `index.html`).                               |
| `nano <archivo>`   | Abre un editor de texto sencillo en la terminal. `Ctrl + X` para salir. |
| `cat <archivo>`    | Muestra el contenido de un archivo de texto en la terminal.             |
| `rm <archivo>`     | Elimina un archivo de forma permanente.                                 |
| `rm -r <carpeta>`  | Elimina una carpeta y todo su contenido.                                |
| `mv <A> <B>`       | **Renombrar:** Cambia el nombre del archivo de A a B.                   |
| `mv <A> <Carpeta>` | **Mover:** Mueve el archivo A dentro de una carpeta.                    |
| `code .`           | Abre la carpeta actual en **Visual Studio Code**.                       |

# 3. Configuración Inicial de Git

Estos comandos se suelen usar una sola vez al instalar Git o al empezar un proyecto nuevo.

|**Comando**|**Qué hace**|
|---|---|
|`git config --global user.name <nombre>`|Configura tu nombre de usuario para los créditos de los cambios.|
|`git config --global user.email <email>`|Configura tu correo electrónico.|
|`git config --list`|Muestra toda tu configuración actual de Git.|
|`git init`|Inicializa la carpeta actual como un repositorio de Git.|
|`git branch -m main`|Cambia el nombre de la rama inicial de `master` a `main`.|

# 4. Flujo de Trabajo y Ramas (Git Workflow)

Aquí es donde ocurre la "magia" del control de versiones: guardar cambios y trabajar en paralelo.

| **Comando**                                        | **Qué hace**                                                                       |
| -------------------------------------------------- | ---------------------------------------------------------------------------------- |
| `git add <archivo>`                                | Sube un archivo específico al **Staging Area** (preparación).                      |
| `git add .`                                        | Añade **todos** los cambios al Staging Area.                                       |
| `git commit -m "mensaje"`                          | Guarda permanentemente los cambios en el historial.                                |
| `git commit --amend`                               | Corrige el último commit (cambiar mensaje o añadir archivos).                      |
| `git rm --cached <archivo>`                        | Quita un archivo del Staging Area pero lo mantiene en tu PC.                       |
| `git log`                                          | Muestra el historial de commits (autor, fecha y hash).                             |
| `git branch`                                       | Muestra las ramas existentes y en cuál estás.                                      |
| `git checkout -b <nombre>`                         | Crea una rama nueva y se cambia a ella automáticamente.                            |
| `git switch <nombre>`                              | Cambia de una rama a otra (o `git checkout <nombre>`).                             |
| `git merge <nombre>`                               | Une los cambios de la rama indicada a la rama donde estás.                         |
| `git branch -D <nombre>`                           | Elimina una rama por completo.                                                     |
| `git reset`                                        | Te devuelve a un commit anterior, eliminando los cambios en el historial.          |
| `git reset --soft`                                 | Solo elimina el archivo del área de staging.                                       |
| `git reset --mixed`                                | Remueve los archivos de staging, manteniendo el historial de commits.              |
| `git reset --hard`                                 | Elimina los archivos y el historial hasta el commit seleccionado.                  |
| `git revert`                                       | Crea un nuevo commit que revierte los cambios realizados por un commit específico. |
| `git tag`                                          | Ver todos los tags.                                                                |
| `git tag -a <nombre del tag> -m <mensaje del tag>` | Crear un tag.                                                                      |
| `git show <nombre tag>`                            | Buscar y ver un tag específico.                                                    |
| `git tag -d <nombre tag>`                          | Para eliminar un tag.                                                              |
