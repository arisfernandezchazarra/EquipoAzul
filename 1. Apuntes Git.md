## 3.1 ¿QUÉ ES UN REPOSITORIO?

En términos simples, un **repositorio** (o "repo") es un espacio de almacenamiento digital donde se guardan todos los archivos de un proyecto, junto con el **historial de cambios** de cada uno de ellos.

Aquí tienes los puntos clave para entenderlo bien:

- **El "Historial de Tiempo":** A diferencia de una carpeta normal en tu computadora, un repositorio registra quién hizo un cambio, qué cambió y cuándo. Esto permite "viajar en el tiempo" y regresar a una versión anterior si algo sale mal.

- **Control de Versiones:** Los repositorios suelen utilizar sistemas como **Git**. Esto permite que varias personas trabajen en los mismos archivos al mismo tiempo sin borrar el trabajo de los demás.

- **Local vs. Remoto:** 
    - __Local:__ Está en tu propia computadora.
    - **Remoto:** Está en la nube (en plataformas como GitHub, GitLab o Bitbucket), lo que permite compartir el código con el mundo o con tu equipo.        

### ¿Para qué se utiliza un repositorio?

Principalmente se asocia al desarrollo de software, pero sirve para cualquier proyecto que requiera orden:

1. **Colaboración:** Permite que un equipo fusione sus avances de forma organizada.
2. **Seguridad:** Tienes una copia de seguridad constante de cada etapa del proyecto.
3. **Documentación:** Cada cambio guardado (llamado _commit_) lleva un mensaje explicando qué se hizo.

### ¿Cómo creamos nuestro primer repositorio?

1. Creamos una nueva carpeta y le damos un nombre (mejor si no tiene espacios).
2. Entramos en la carpeta, botón derecho y le damos a __Abrir en Terminal__ o darle a __Mostrar más opciones__ y buscar __Open GIT Bash here__ si tenemos instalado __GIT__.

## 3.2 ¿QUÉ ES GIT?

Si el **repositorio** es el "almacén", **Git** es el "conserje inteligente" que gestiona todo lo que ocurre dentro. **Git** es un sistema de **control de versiones**. Su trabajo principal es vigilar los cambios que haces en tus archivos para que nunca pierdas nada.

### ¿Cómo empezamos a utilizar Git?

1. Descargamos [__Git__](https://git-scm.com/install/windows) y lo instalamos. Nos dará la opción para elegir en qué editor queremos visualizarlo. Elegimos Visual Studio Code.
2. Descargamos [Visual Studio Code](https://code.visualstudio.com/download) y lo instalamos.

## 3.3 EL PROBLEMA DE LOS ESPACIOS EN LOS NOMBRES

Esto es vital: **A la terminal no le gustan los espacios.**

Si tienes una carpeta llamada `Mis Proyectos`, la terminal pensará que `Mis` es un comando y `Proyectos` es otra cosa.

- **Solución 1:** Usa comillas: `cd "Mis Proyectos"`.
- **Solución 2 (Recomendada):** No uses espacios. Usa guiones o barras bajas: `mis_proyectos` o `mis-proyectos`.

## 3.4 TABLA DE COMANDOS BÁSICOS

Estos comandos se usan en la **Terminal** (o Consola).

| **Comando**                                                   | **Qué hace**                                                                                                                                                                                            |
| ------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `git version`                                                 | Te dice qué versión de Git tienes instalada.                                                                                                                                                            |
| `pwd`                                                         | (_Print Working Directory_) Te dice en qué carpeta estás ahora mismo.                                                                                                                                   |
| `ls`                                                          | (_List_) Te muestra todos los archivos y carpetas de donde estás.                                                                                                                                       |
| `ls -R`                                                       | _(List Recursive)_ Te muestra todos los archivos y carpetas de forma recursiva. Muestra los archivos u las carpetas dentro de cada carpeta de la carpeta en la que estás.                               |
| `cd <nombre carpeta>`                                         | Entra en una carpeta. Si tiene espacios: `cd "nombre carpeta"`. Si empiezas a escribir el nombre de la carpeta y pulsas tabulador te completa el nombre de la carpeta. ___Por ejemplo: cd Des...ktop___ |
| `cd ..`                                                       | Sale de la carpeta actual y te sube un nivel hacia atrás.                                                                                                                                               |
| `cd ~`                                                        | (Equivalente a `cd --`) Te lleva a la carpeta raíz de tu usuario (el origen).                                                                                                                           |
| `mkdir nombre_carpeta`                                        | Crea una carpeta nueva con ese nombre.                                                                                                                                                                  |
| `mkdir -p <nombre carpeta>/<nombre carpeta>/<nombre carpeta>` | Crear múltiples carpetas, una dentro de otra.                                                                                                                                                           |
| `git init`                                                    | Convierte la carpeta actual en un repositorio de Git (empieza a vigilar cambios).                                                                                                                       |
| `git config --global user.name <nombre de usuario >`          | Configura o muestra el **nombre** de usuario.                                                                                                                                                           |
| `git config --global user.email <e-mail>`                     | Configura o muestra el **email** de usuario.                                                                                                                                                            |
| ``git config --list``                                         | Para ver la configuración en Git.                                                                                                                                                                       |
| `rm -r <nombre carpeta>`                                      | Borrar directorio y todo su contenido.                                                                                                                                                                  |
| `rm <nombre archivo>`                                         | Borrar archivo.                                                                                                                                                                                         |
| `touch <nombre archivo.txt>`                                  | Crear documento de texto.                                                                                                                                                                               |
| `nano <nombre archivo.txt>`                                   | Escribir dentro del archivo de texto (si existe). Si no existe documento de texto lo crea automáticamente con el nombre de archivo que le pongamos. Ctrl + X para cerrar y guardar.                     |
| `cat <nombre archivo.txt>`                                    | Ver qué contiene el archivo de texto.                                                                                                                                                                   |
| `Ctrl + C`                                                    | Cortar cualquier proceso que se arranque en la terminal.                                                                                                                                                |
| `mv <archivo_A> <archivo_B>`                                  | **Renombrar:** El archivo se queda donde está, pero cambia su nombre de A a B.                                                                                                                          |
| `mv <archivo_A> <carpeta_X>`                                  | **Mover:** El archivo mantiene su nombre pero se guarda dentro de la carpeta X.                                                                                                                         |
| `mv <archivo_A> <carpeta_X>/<archivo_B>`                      | **Mover y Renombrar:** El archivo se va a la otra carpeta y además cambia su nombre.                                                                                                                    |
| `code .`                                                      | Abrir __Visual Studio Code__ desde la consola de __Git Bash__                                                                                                                                           |
| `git branch -m main`                                          | Cambiar nombre de __master__ a __main__                                                                                                                                                                 |
| `git add <archivo>`                                           | Sube un archivo específico al "escenario" (Staging Area). Solo este archivo entrará en el próximo commit.                                                                                               |
| `git add .`                                                   | Añade **todos** los cambios (nuevos, editados o borrados) de la carpeta actual al Staging Area.                                                                                                         |
| `git commit -m <mensaje>`                                     | Guarda los cambios que están en el escenario de forma permanente en el historial con un mensaje descriptivo.                                                                                            |
| `git commit --amend`                                          | Sirve para arreglar el último commit realizado (por si olvidaste un archivo o quieres cambiar el mensaje).                                                                                              |
| `git rm --cached <nombre archivo>`                            | Quitar un archivo del Staging Area.                                                                                                                                                                     |
| `git log`                                                     | Ver el registro de todos los __commits__ que hemos hecho. Junto a __commit__ aparece un código muy largo (___hash__ / DNI o huella dactilar de ese commit_), el autor y la fecha.                       |
| `git branch`                                                  | Para ver en qué rama me encuentro.                                                                                                                                                                      |
| `git checkout -b <nombre rama>`                               | Para crear una rama nueva.                                                                                                                                                                              |
| `git checkout (o git switch) main`                            | Para volver a la rama principal.                                                                                                                                                                        |
| `git checkout (o git switch) <nombre rama>`                   | Para ir de una rama a otra rama.                                                                                                                                                                        |
| `git merge <nombre rama>`                                     | Para unificar las ramas desde la rama principal.                                                                                                                                                        |
| `git branch -D <nombre rama>`                                 | Para eliminar una rama.                                                                                                                                                                                 |

__Un ejemplo rápido de flujo:__
```
1. Abres la terminal.
2. `mkdir mi-proyecto` (Creas carpeta).
3. `cd mi-proyecto` (Entras en ella).
4. `git init` (¡Listo! Git ya está trabajando aquí).
5. `git config --global user.name arisfernandezchazarra` (Configuras el nombre usuario)
6. `git config --global user.email arisfernandezchazarra@gmail.com` (Configuras el email)
7. `git config --list` (Ves que la configuración está bien)
```

__Sintaxis de código para entenderlo:__
`git (actor) ➜ commit (verbo) ➜ -m (condición o flag) ➜ < mensaje >`
